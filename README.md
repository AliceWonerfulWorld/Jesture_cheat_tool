# Jesture Show Control

`PRODUCT_BLUEPRINT.md` を元にした、カメラ前の手ジェスチャーで大きな表示ボードを操作するジェスチャーゲーム用の静的 Web アプリです。

## 機能

- 6ジャンルのカテゴリー選択
- ジャンルごとのお題カード選択
- お題は常に1つだけ選択
- マウス / タッチ操作
- デフォルトONのカメラと MediaPipe Hands による手カーソル
- グーのホールドによる決定操作
- 片手チョキの戻る操作
- ホールド時間とカーソル追従の調整
- カメラプレビューの表示切り替え
- 全画面表示
- Web Audio API の短い効果音
- PeerJS + WebRTC DataChannel のルーム同期

## 起動

```bash
python3 -m http.server 8010
```

ブラウザで開きます。

```text
http://127.0.0.1:8010/
```

カメラ機能は `localhost` または HTTPS の URL で動きます。CDN から MediaPipe Hands と PeerJS を読み込むため、初回利用時はインターネット接続が必要です。

## 使い方

- `カメラON`: カメラと手検出の状態
- `Sound ON`: 効果音を有効化
- `Sender`: 操作端末としてルームを開く
- `Viewer`: 表示端末として `?room=xxxxxx&role=viewer` に接続
- `Solo`: 同期なしで操作
- `Copy URL`: 表示端末用 URL をコピー
- `Hold`: グーを維持して決定するまでの時間
- `Smooth`: 手カーソルの追従速度
- `Camera preview`: カメラ映像と骨格表示の表示切り替え
- `前へ` / `次へ`: 大量のお題を12件ずつページ送り

カード上でグーを約 0.7 秒ホールドするとお題を1つだけ選択できます。回答が終わったら、片手でチョキを作るとカテゴリー選択へ戻れます。クリック操作でも同じ画面遷移を確認できます。

## 同期プレイ

1. 出題側の端末で `操作` を選びます。
2. `URLコピー` で表示端末用 URL をコピーして、スマホなどに送ります。
3. 表示端末側は、その URL を開くと `表示` モードでルームに接続します。
4. 出題側がカテゴリーを選ぶと、表示端末には「お題選択待ち」が表示されます。
5. 出題側がお題を選ぶと、表示端末には選ばれたお題だけが大きく表示されます。
6. 出題側がカテゴリー選択に戻ると、表示端末も待機画面に戻ります。

ローカル環境でスマホから開く場合、`127.0.0.1` や `localhost` はスマホ自身を指します。同じWi-Fi上で使うときは、表示端末用URLのホスト名を出題側PCのIPアドレスに置き換えてください。

表示端末が送信側より先に開かれた場合は、自動で再接続します。ルームIDが残っていて使えない場合は、出題側が新しいルームIDを自動作成します。
